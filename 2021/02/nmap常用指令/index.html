NMAP常用指令

Nmap 常用指令

  • nmap -sT 192.168. xxx.xxx   // TCP连接扫描，不安全(可能会被记录下来)，慢

  • nmap -sS 192.168. xxx.xxx   // SYN标记扫描(半开连接,只使用了前两次握手),使用最频繁，安全，快

    回复标志 状态
    SYN/ACK 开放
    RST/ACK 关闭
    ICMP unreachable 屏蔽

  • TCP 标志位(若目标主机回复了RST(重置连接)标志位，表示该端口关闭；若无回应，则该端口处于打开|过滤状态)

    • nmap -sN 192.168.xxx.xxx // 不设置标志位
    • nmap -sF 192.168.xxx.xxx // 设置FIN标志位
    • nmap -sX 192.168.xxx.xxx // 扫描设置FIN、PSH和URG标识位

  • nmap -Pn 192.168. xxx.xxx   // 目标机禁用ping，绕过ping扫描

  • nmap -sU 192.168. xxx.xxx   // UDP扫描,慢,可得到有价值的服务器程序

    • -p 端口范围：后接指定端口,若扫描1~2024号端口,则用 -p 1-2024
    • -F（快速扫描）：仅扫描常用的100个端口
    • -r（顺序扫描）：按照从小到大的顺序进行端口扫描

  • nmap -sI 僵尸ip 目标ip   // 使用僵尸机对目标机发送数据包

  • nmap -sA 192.168. xxx.xxx   // 检测哪些端口被屏蔽

  • nmap 192.168. xxx.xxx -p   // 对指定端口扫描

  • nmap 192.168.96.1/24   // 对整个网段的主机进行扫描

  • nmap 192.168. xxx.xxx -oX myscan.xml   // 对扫描结果另存在myscan.xml

  • nmap -T1~6 192.168. xxx.xxx   // 设置扫描速度，一般T4足够

  • nmap -sV 192.168. xxx.xxx   // 对端口上的服务程序版本进行扫描

  • nmap -O 192.168. xxx.xxx   // 对目标主机的操作系统进行扫描

  • nmap -sC 192.168. xxx.xxx   // 使用脚本进行扫描，耗时长

  • nmap -A 192.168. xxx.xxx   // 强力扫描，耗时长

  • nmap -6 ipv6地址   // 对ipv6地址的主机进行扫描

  • nmap -f 192.168. xxx.xxx   // 使用小数据包发送，避免被识别出

  • nmap –mtu 192.168. xxx.xxx   // 发送的包大小,最大传输单元必须是8的整数

  • nmap -D <假ip> 192.168. xxx.xxx   // 发送参杂着假ip的数据包检测

  • nmap –source-port   // 针对防火墙只允许的源端口

  • nmap –data-length: 192.168. xxx.xxx   // 改变发生数据包的默认的长度，避免被识别出来是nmap发送的

  • nmap -v 192.168. xxx.xxx   // 显示冗余信息(扫描细节)

  • nmap -sn 192.168. xxx.xxx   // 对目标进行ping检测，不进行端口扫描（会发送四种报文确定目标是否存活,）

  • nmap -sP 192.168. xxx.xxx   // 仅仅对目标进行ping检测

  • nmap -n/-p 192.168. xxx.xxx   // -n表示不进行dns解析，-p表示要

  • nmap –system-dns 192.168. xxx.xxx   // 扫描指定系统的dns服务器

  • nmap –traceroute 192.168. xxx.xxx   // 追踪每个路由节点

  • nmap -PE/PP/PM   // 使用ICMP echo, timestamp, and netmask 请求包发现主机

  • nmap -sP 192.168. xxx.xxx   // 主机存活性扫描，arp直连方式

  • nmap -iR [number]   // 对随机生成number个地址进行扫描
